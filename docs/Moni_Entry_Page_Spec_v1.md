## 1. 页面角色与设计哲学

### 1.1 角色定位

记账页是**工具入口**，不是主舞台。它在底部三栏导航的右侧位置，与首页和设置页并列。

记账页承载两个不同性质的工作流：

1. **数据导入**：低频但重要（典型频率：每周一次）
2. **随手记**：高频日常动作（现金消费、补漏场景）

### 1.2 设计哲学

**"拖一笔"而不是"填一表"**

Moni 的随手记不做成传统的表单页。传统记账 App 的表单交互（打开页面 → 逐项填写 → 点保存）是功能性的但无趣的。Moni 选择复用首页已经建立的"拖拽归类"交互母题：

- 首页：把**已有条目**拖进分类 → 教会 AI
- 记账页：把**一笔新记录**拖进分类 → 创建条目

用户只需要学一次手势，就能在两个场景中自然使用。这个交互母题是 Moni 的品牌记忆点之一。

### 1.3 视觉气质

- 比首页**更安静**：Memphis 装饰密度降低约 40%
- 背景保持暖米白 `#F5F0EB`
- 留更多呼吸空间，不堆叠信息
- 数据导入卡是视觉亮点，设计目标是"治愈感"——降低用户对"导入操作复杂"的心理负担

## 2. 页面信息架构

从上到下固定为：

1. **Header**（与首页共享，粘性固定）
2. **页面标题区**（"记账" + 副标题）
3. **数据导入卡**（粗边框 Memphis 卡片）
4. **随手记区域**（标题 + 操作说明 + 最近流水参考区 + 记录计数）
5. **"记一笔"浮动按钮**（固定在底部导航上方）
6. **底部三栏导航**（记账图标为激活态）

页面整体可纵向滚动，但在 860px 标准幕布内通常无需滚动。

## 3. Header

### 3.1 结构

与首页完全共享同一个 Header 组件：

- 左：Moni 字标
- 右：账本选择器（显示当前账本名）

### 3.2 规则

- Header 是粘性元素，始终在顶部
- 记账页不新增 Header 内的额外元素
- 账本选择器显示的是当前活跃账本，随手记写入的就是这个账本

## 5. 数据导入卡

### 5.1 设计目标

数据导入（从微信/支付宝导出 CSV 账单）是用户负担较高的操作。导入卡的设计目标是：

1. **降低心理门槛**：视觉上传达"这件事很简单、很值得做"
2. **提供教程入口**：用户不知道怎么在微信/支付宝里导出时，能立刻找到帮助
3. **视觉治愈感**：用 Memphis 几何印花 + 柔和配色创造温暖友好的氛围

### 5.2 结构

导入卡是一张粗边框卡片，整体延续 Memphis 风格与治愈感。其静态布局允许在实现中微调，但应满足以下结构关系：

**主体区**：
- 需要包含导入主题插画 / 图标、标题文案、说明文案
- 允许采用单一主按钮或来源分流按钮（如微信 / 支付宝）两种实现形式
- 文案语义保持为"导入账单数据"、"从微信或支付宝导出账单，让 AI 帮你自动整理"这一类口径，不强制逐字一致

**教程入口区**：
- 保留独立教程入口
- 视觉上与主体区形成层级分隔
- 文案保持"不知道怎么导出？查看导入指南 / 图文教程"这一类帮助性表达

### 5.3 视觉规则

- 卡片内部有 Memphis 几何印花（程序生成，密度低，不遮挡文字）
- 印花使用品牌色系，opacity 在 0.10-0.24 之间
- 图标 / 插画区域延续薄荷绿、天蓝、暖米白等低饱和品牌色系

### 5.4 交互规则

- 点击"导入"按钮 → 进入数据导入流程（本文不展开，后续独立规格）
- 点击教程入口 → 打开图文教程页面（外部链接或内嵌 WebView，本文不展开）
- 整个卡片不做为点击区域，只有按钮和教程链接可点击

### 5.5 术语说明

- UI 上统一称为"数据导入"或"导入账单数据"，不使用"CSV 导入"
- 用户不需要知道文件格式是 CSV

## 6. 随手记区域

### 6.1 区域结构

随手记区域在导入卡下方，包含：

1. **区域标题**："随手记"
2. **轻量场景提示**：用于说明这是现金消费、补漏记录等快速入口
3. **最近流水参考区或空态区**：用于提供最近消费语境，并帮助建立反馈闭环
4. **"记一笔"主操作按钮**

### 6.2 随手记区域静态执行说明

- 该区域静态布局以参考 JSX 与最终实现为准
- 当前收敛口径为"最近流水 / 空态卡片"方案，不再以分类预览网格作为默认静态表达
- 不论采用哪种静态方案，都不能改变后续长按拖拽进入分类投放的动态主路径
- 整体气质应比首页更克制、更安静，不制造过强视觉竞争

### 6.2.1 最近流水参考区规则

- 最近流水区展示的是**首页最新一天中的最新两条记录**
- 该区作为信息参考区，不区分来源类型（微信 / 支付宝 / 手动）
- 该区**不受首页 data range 约束**
- 该区不承担严格统计含义，也不是本页独立的数据过滤结果
- 若最新一天暂无可展示记录，则回退为空态区
- 提示小字与"记一笔"按钮属于最近流水卡片后的同级内容，应紧贴卡片下方，不单独悬浮在页面中段

### 6.3 记录计数

- 当用户在本次访问中已成功记录 ≥ 1 笔时，显示一条轻量提示条
- 样式：薄荷绿浅底 + 绿色文字，"📝 本次已记录 N 笔"
- 无记录时此区域不显示

## 7. "记一笔"按钮

### 7.1 静态态

- **位置**：位于页面下半部分居中，提示小字与按钮整体应紧贴最近流水卡片下方；同时仍需与底部首页 icon 保持明确间距，避免视觉上贴得过近
- **宽度方向**：不是全宽按钮，应保持明显的悬浮感与可拖拽感；结合当前修订意见，静态态按钮应比先前版本**更收窄**
- **样式**：深底色 `#222`，浅米白文字，圆角粗边框，保留轻微阴影
- **内容**：铅笔图标 + "记一笔" 文字
- **品牌装饰**：保留 Moni 品牌三色装饰，但不冻结具体坐标；装饰应服务于品牌识别，不可喧宾夺主

### 7.1.1 本轮静态口径补充

- 针对当前首页拖拽体验，静态态"记一笔"按钮不应过宽，避免在长按收缩前显得过重
- 静态态按钮应处于页面下半部分的稳定主操作位，不与底部导航中央首页 icon 抢位
- 长按进入拖拽后，按钮收缩为一个**黑色、略长方形、带装饰的铅笔悬浮 token**
- 收缩后"记一笔"文字消失，只保留铅笔图标
- 收缩后背景色保持黑色，不改为白色或浅橙色

### 7.2 长按启动

- **触发方式**：长按（400ms 阈值）
- **触发后行为**：
  1. 按钮执行**收缩动画**：从当前尺寸 → 48×40 的小方块
  2. "记一笔"文字消失，替换为铅笔图标（居中）
  3. 背景保持深色 `#222`
  4. 品牌三色装饰缩小但保留，位于右上角
  5. 整个小方块**脱离原位**，跟随用户手指移动
  6. 同时，全屏拖拽蒙版淡入

### 7.3 拖拽态

- 小方块样式：48×40，borderRadius 12，background `#222`，boxShadow `0 6px 24px rgba(0,0,0,.3)`
- 小方块位置：`position: fixed`，跟随手指，transform `translate(-50%, -115%)`（悬浮在手指上方）
- 小方块内容：铅笔图标（18×18，米白色笔触）+ 右上角品牌三色装饰
- 小方块与首页拖拽条目的区别：首页拖拽的是白底卡片（显示商户名+金额），这里拖拽的是深色品牌 token（只有铅笔图标）

### 7.4 收缩动画规格

```
持续时间：250ms
缓动函数：cubic-bezier(.34, 1.56, .64, 1)  // 带微弹的减速
动画属性：width, height, padding, border-radius
文字消失：opacity 0 → 铅笔图标 opacity 1（交叉淡入，同步执行）
```

## 8. 拖拽蒙版

### 8.1 与首页蒙版的关系

记账页的拖拽蒙版**复用首页 DragOverlay 的视觉设计与交互逻辑**。差异点：

| 维度 | 首页 DragOverlay | 记账页蒙版 |
|------|-----------------|-----------|
| 触发方式 | 长按条目 | 长按"记一笔"按钮 |
| 悬浮 token | 白底卡片（商户名+金额） | 深色方块（铅笔图标+品牌装饰） |
| 标题文案 | "拖放到分类中" | "拖放到分类中，开始记一笔" |
| 投放后行为 | 弹出 ReasonDialog（输入修正理由） | 弹出 EntryPanel（输入金额等字段） |

当前口径补充：

- 蒙版除顶部描述文案外，背景、标题层级、分类网格布局、hover 高亮与关闭方式统一按首页 DragOverlay 范式执行
- 顶部描述文案保持记账页自己的表达："拖放到分类中，开始记一笔"
- 记账页只保留自己的悬浮 token 差异，不再额外发明第二套蒙版视觉语言

### 8.2 蒙版结构

- 全屏覆盖，background `rgba(0,0,0,.55)`
- 顶部：标题文案（白色，fontSize 14，fontWeight 700，居中）
- 中部：分类投放网格，覆盖全部 11 个分类
- 每个分类格子：白底，2.5px 边框（默认浅灰，hover 时变为分类主色），borderRadius 12，内含 emoji（20px）+ 分类名（12px）

### 8.2.1 蒙版静态布局说明

- 分类投放网格的具体列数、gap、上下留白可根据实现微调
- 只要保证手指移动时目标清晰、命中面积充足、11 个分类可稳定浏览与投放即可
- 蒙版静态布局以参考 JSX 与最终实现为准，不再冻结过细尺寸

### 8.3 命中检测

- 使用 `document.elementFromPoint(clientX, clientY)` 实时检测手指下方的分类格子
- 检测逻辑需要向上查找 `[data-drop-category]` 属性（与首页一致）
- 命中时：格子边框变为分类主色 + `scale(1.05)` 放大 + transition 200ms
- 离开时：恢复默认

### 8.4 投放

- 手指在命中某个分类格子的状态下松手 → 触发投放
- 蒙版淡出（200ms）
- 弹出二级录入面板（见第 9 节）
- 手指不在任何分类格子上松手 → 取消拖拽，蒙版淡出，回到静态态

### 8.5 指针捕获

- 参照 `DESIGN.md` §23.2 的指针捕获规范
- 拖拽启动后，指针事件必须在 `window` 级别监听（`pointermove`、`pointerup`、`pointercancel`）
- 不依赖单个元素的 `onPointerMove`
- 拖拽期间锁定 body 滚动

## 9. 二级录入面板（EntryPanel）

### 9.1 触发条件

用户将"记一笔" token 投放到某个分类格子后弹出。

### 9.2 面板结构

居中模态对话框，从中心弹入（`scale .85 → 1`，300ms，带微弹缓动）。

从上到下：

**① 分类标示行**
- 左：分类图标容器（42×42，分类浅底色背景 + emoji）
- 中："记录到" 小字 + 分类名大字（分类主色）
- 右：收支方向切换 pill（"支出" / "收入"，默认"支出"）

**② 金额输入区（Hero）**
- 浅底色容器（`#FAFAF8`），1.5px 边框，borderRadius 14
- 小字标签"金额"
- 大字号数字输入：`¥` 前缀 + input（fontSize 32，fontWeight 700，Space Mono 等宽字体）
- 自动聚焦（autoFocus），弹起数字键盘（`inputMode="decimal"`）

**③ 可选字段区**
- 三个输入行，纵向排列，gap 8：
  - 主题（subject）：placeholder "这笔花在哪了？（选填）"
  - 描述（description）：placeholder "补充说明（选填）"
  - 日期：默认显示"今天 HH:mm"，可点击修改
- 样式统一：padding 10px 12px，borderRadius 10，1.5px 浅灰边框，fontSize 13

**④ 保存按钮**
- 全宽，padding 13px，borderRadius 12
- 金额有效时：深底色 `#222`，米白文字
- 金额为空或 ≤ 0 时：灰色 `#CCC`，不可点击
- 文案："保存"

### 9.3 收支方向切换

- 双 pill 切换：左"支出"（珊瑚红），右"收入"（薄荷绿）
- 激活态：对应颜色填充底，白色文字
- 非激活态：白底，灰色文字
- 默认值：支出
- **跨记录保持**：保存后重新进入面板时，方向保持上次选择（不重置）

### 9.4 字段映射

面板字段与 `Moni_Manual_Entry_Spec_v3.md` §2.3 `ManualEntryInput` 的映射：

| 面板元素 | 映射字段 | 必填 |
|---------|---------|------|
| 金额输入 | `amount` | ✅ |
| 收支切换 | `direction` | ✅（有默认值） |
| 投放的分类 | `category` | ✅（拖拽决定） |
| 主题输入 | `subject` | ❌ |
| 描述输入 | `description` | ❌ |
| 日期选择 | `date` | ❌（默认当前时刻） |

### 9.5 关闭面板

- 点击蒙版背景 → 关闭面板，回到静态态（放弃本次录入）
- 不提供关闭按钮（点击背景即可关闭，减少视觉噪音）
- 关闭时面板执行反向动画（`scale 1 → .85`，opacity → 0，200ms）

## 10. 保存反馈

### 10.1 校验

保存前校验：

1. `amount` > 0 → 否则按钮灰色不可点击
2. `category` 已选（由拖拽投放保证）→ 始终满足

校验通过后立即执行保存动作。

### 10.2 成功动效

保存后，EntryPanel 过渡为成功确认态：

1. 面板内容替换为居中的成功卡片（弹入动画 300ms）
2. 成功卡片内容：
   - 大号对勾 "✓"（fontSize 36）
   - "已记录"（fontSize 16，fontWeight 700）
   - 金额显示（Space Mono，fontSize 22，支出用珊瑚红 / 收入用薄荷绿）
   - 分类图标 + 分类名
3. 成功卡片**停留 1.2 秒**后自动关闭
4. 关闭后：蒙版消失，回到记账页静态态

### 10.3 表单重置

成功保存后，以下状态重置：

| 状态 | 重置行为 |
|------|---------|
| 金额 | 清零 |
| 分类 | 清空（下次需重新拖拽选择） |
| 主题 | 清空 |
| 描述 | 清空 |
| 日期 | 恢复为"当前时刻" |
| **收支方向** | **不重置**，保持上次选择 |

### 10.4 记录计数

页面维护一个 `successCount` 计数器（页面级状态，非持久化），每次保存成功 +1。计数 > 0 时在随手记区域下方显示提示条。

## 11. 底部导航

### 11.1 与首页的关系

记账页与首页共享同一个 BottomNav 组件。差异点：

| 维度 | 首页 | 记账页 |
|------|------|--------|
| 激活图标 | 中央品牌按钮 | 右侧"记账"图标 |
| 中央按钮功能 | AI 状态 + 控制（长按） | 导航回首页（单击） |
| 右侧图标 | "记账"（灰色描边） | "记账"（深色描边 + 粗体文字） |

### 11.2 图标规则

- 设置图标：灰色描边，与首页一致
- 中央品牌按钮：与首页视觉一致，但在记账页不承载 AI 控制功能，单击导航回首页
- 记账图标：在记账页为激活态——描边色变深（`#222`），标签文字变 fontWeight 700 + color `#222`

## 12. Memphis 视觉规则

### 12.1 页面背景装饰

- 使用程序生成的几何印花（与首页同一套 `seededShapes` 函数）
- 种子值与首页不同（记账页使用 seed 333，首页使用 777），保证图案不重复
- 密度：约 7 个形状，比首页（9 个）略少
- 区域：覆盖整个页面背景，从 y=40 到 y=750
- 不遮挡文字、按钮、卡片内容

### 12.2 数据导入卡装饰

- 卡片内有独立的 Memphis 印花层（seed 42，5 个形状）
- 形状覆盖整张卡片，但 opacity 较低（0.10-0.24）
- 不干扰卡片内的文字和按钮

## 13. 手势交互规范

### 13.1 全局 CSS 防御层

继承 `DESIGN.md` §23.1 的全部 6 条 CSS 规则，不做例外。

### 13.2 长按阈值

"记一笔"按钮的长按激活阈值为 **400ms**。

理由：首页长按拖拽条目也是 400ms，保持一致。

### 13.3 拖拽期间的全局锁定

拖拽启动后：

1. `body` 滚动锁定
2. 所有 pointer 事件在 `window` 级别捕获
3. 拖拽结束（投放或取消）后恢复

### 13.4 取消拖拽的方式

1. 在非分类区域松手 → 取消
2. `pointercancel` 事件 → 取消
3. 取消后：蒙版淡出，"记一笔"按钮恢复静态态
