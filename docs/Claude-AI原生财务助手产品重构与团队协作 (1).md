## 一、从你的发言中提炼的思维模型

通读了整段对话中你的所有 Prompt，你的产品设计思维有几个非常鲜明的特征：

**穷举后收敛**——你从不接受"大概这样就行"。每到一个设计决策点，你的本能反应是"把所有情况都列出来"，然后逐个确认。标签变更的四种场景、重分类的六种触发、实例库的每条入口/出口——你不是在做设计，你是在建一张无死角的状态矩阵。

**用户摩擦的极端敏感**——你反复出现的句型是"不要弹窗"、"不要一次性给用户弹那么多选项"、"用户改完标签就完了，别打扰他"。你的直觉是：每多一次点击都是产品在向用户借债，债务不可积累。

## 二、产品哲学（最高层次）

从你的思维方式中蒸馏出的产品哲学，不是我替你发明的，而是你散布在各处对话中已经说过的话的提纯：

### 哲学一：修正即教学，无需第二个动作

用户改一个分类标签——这是系统获得学习信号的唯一渠道，也是用户需要做的唯一一件事。实例采集、记忆更新、重分类调度全部自动发生。用户甚至不需要知道"AI 在学习"这件事。

### 哲学二：AI 自治但人可兜底

AI 自主决定记忆内容、自主提炼规则、自主压缩冗余。但用户随时可以打开"AI 的笔记本"看到它学了什么、改掉它学错的、回退到任意历史版本。信任是默认态，但安全阀永远畅通。

### 哲学三：渐进式披露，绝不一次性堆砌

任何操作的后果都通过层层递进的方式呈现。删了标签？先问"要不要重分类"。要？再问"什么范围"。选了全量？再展示锁定列表让你检查。三层对话，每层只问一个问题。

### 哲学四：数据即界面，数据也即学习素材

流水列表不只是展示层——它同时是用户修正分类的操作层，也是 AI 采集学习信号的数据层。同一条交易记录，同时服务于展示、交互、和机器学习三个职责，零冗余。

### 哲学五：结构层稳定，内容层呼吸

界面骨架不因数据加载而跳动。数据到来是"容器被内容激活"，不是界面重建。AI 在后台的工作不改变界面的结构，只改变数据的颜色和状态。

## 三、核心用户体验定义

### 3.1 主线体验闭环

```
导入 CSV → AI 自动分类 → 用户浏览流水 → 发现错误 → 改一下标签 →
```

整个闭环中，用户的意识只停留在"我在看账单、改错误"，而不是"我在训练 AI"。

### 3.2 关键交互节点

**节点 A：分类状态的环境感知**——AI 正在工作时，列表边缘有微妙的光效流动（正常分类绿色，重分类黄色）。完成时光效消散，不弹窗、不打断。

**节点 B：修正即完成**——用户改完标签的那一瞬间，一切就结束了。底部可能浮现一条极轻的提示"想告诉 AI 为什么吗？"，3 秒后自动消失。用户可以忽略，也可以说一句话。

**节点 C：AI 记忆的透明性**——设置页中，用户可以看到一个有序列表，那是 AI 学到的所有东西。每条都是自然语言，一句话一个信息点。用户可以改、可以删、可以回退到任意历史快照。

**节点 D：标签是知识锚点**——新增标签时必须写一句描述。这不是表单仪式，这是给 AI 的冷启动种子——没有这句话，AI 对新标签一无所知。

## 四、初版 UI 视觉风格建议（面向比赛 / 移动端）

既然初版只求快速落地、不追求动画细节，以下是视觉层面的核心建议：

### 4.1 风格定位：轻量孟菲斯（Light Memphis）

不是完全复刻孟菲斯那种"几何爆炸"，而是**取其明亮色调和几何装饰性，降低其视觉密度**，让金融数据保持可读性。

**保留**：浅色/米白背景、高饱和彩色几何装饰（圆、三角、半圆、波浪线）、粗描边卡片
**降低**：密集的点阵纹理、过多的撞色、满屏装饰元素
**新增**：大面积留白、清晰的信息层次、数据区域的呼吸感

### 4.2 色彩方案

以参考图为基底：米白背景（#F5F0EB 附近）+ 五色装饰体系。数据信息用中性色（深灰/黑），彩色只用于装饰元素和状态标识。

支出用珊瑚红（参考图中的粉红色系），收入用薄荷绿（参考图中的青绿色系）。分类标签可以分配不同的几何色块作为视觉标识——这恰好和 Memphis 的多色几何完美适配。

### 4.3 字体策略

标题/品牌：一款有个性的英文无衬线（如 Space Grotesk 或 DM Sans），搭配中文黑体。
数据/正文：等宽字体保持金额对齐（Space Mono 或 JetBrains Mono）。
不再使用像素字体——那是旧视觉概念的遗产。

### 4.4 组件语言

卡片：圆角 + 粗描边（2-3px 黑色边框），内部用白色或极浅色底。
按钮：实心色块 + 粗描边，hover 时几何位移（不用阴影，用位置偏移表达深度）。
标签 Tag：小胶囊形，每个分类对应一个几何形状 + 颜色组合（如 meal = 圆形 + 珊瑚红，transport = 三角形 + 蓝色）。
装饰元素：页面角落和空白区散布几何图形（半圆、波浪线、点阵小块），密度低于参考图，保持金融工具的专业感。

| **AI 刚学完** | 学习会话刚完成 | 流水列表顶部浮现："AI 学到了新偏好"，点击查看记忆变化 |
| **全部分好了** | 0 笔未分类 | 摘要区权重增大，展示"本周 / 本月"统计亮点 |

这不是 5 个页面，是**同一个页面的 5 种呼吸状态**。布局骨架不变——永远是"顶部摘要 + 下方流水"——但摘要区的内容和视觉权重会根据情景自动调整。用户不需要思考"我该去哪个页面"，因为答案已经在眼前了。

## 流水列表的重新设计：从表格到卡片流

### 设计原则：让数据讲故事，让错误自己跳出来

旧版每条交易是一个紧凑的行，所有信息等权重排列。新版要做的改变：

**一天是一张卡片，不是一个分组头。**

```
┌──────────────────────────────────────────┐
│                                          │
│  ┌─ 昨天 · 周六 ──────── 支出 ¥187 ───┐│
│  │                                      ││
│  │  🍜 杨国福麻辣烫            ¥45     ││
│  │     正餐 · 18:10 · 微信              ││
│  │                                      ││
│  │  🚗 滴滴出行                ¥23     ││
│  │     交通 · 16:30 · 支付宝            ││
│  │                                      ││
│  │  🧋 益禾堂                  ¥12     ││
│  │     ⚠️ 未分类 · 19:40 · 微信         ││  ← 未分类项用警示色
│  │                                      ││
│  └──────────────────────────────────────┘│
│                                          │
│  ┌─ 周五 4/4 ──────────── 支出 ¥256 ──┐│
│  │  ...                                 ││
│  └──────────────────────────────────────┘│
│                                          │
└──────────────────────────────────────────┘
```

**卡片化的好处**：

每天的消费自然形成一个视觉单元，用户一眼能感知"这天花了多少"。日卡片的描边和背景可以做微妙的情绪映射——花得少的日子平静（浅色边框），花得多的日子热烈（暖色边框或几何装饰更密集）。Memphis 风格下，每张日卡片可以有微妙的装饰差异（不同角落的几何点缀），让滑动浏览变得不单调。

**交互：点击条目不跳转详情页，而是原位展开。**

这是从你旧版设计哲学中继承的"同源生长"——用户点击一条交易，这条交易就在原位舒展开，露出完整信息和操作区：

```
│  🍜 杨国福麻辣烫            ¥45.00    │
│  ┌─ 展开区 ────────────────────────┐  │
│  │                                  │  │
│  │  分类: [正餐 ▼]   🔒 锁定       │  │
│  │                                  │  │
│  │  AI: "餐饮商户，正餐时段"        │  │
│  │                                  │  │
│  │  告诉AI为什么改:                 │  │
│  │  ┌──────────────────────────┐   │  │
│  │  │                          │   │  │
│  │  └──────────────────────────┘   │  │
│  │                                  │  │
│  │  微信 · 零钱 · 餐饮美食          │  │
│  │                                  │  │
│  └──────────────────────────────────┘  │
```

**为什么不跳转**：跳转是一次上下文切换。用户要看的东西（这条交易在这一天的消费流中的位置）消失了。原位展开保持了上下文——用户可以看到上面一条和下面一条是什么，帮助他判断"这笔该怎么分"。

改完分类后，展开区可以自动折叠（0.5s 延迟 + 动画），也可以手动折叠。下一条未分类的条目可以轻微抖动或脉动，暗示"接下来看看我？"——但不强制，不弹窗。

**未分类条目的视觉待遇**：

它们不只是换个颜色标签。它们应该有一种**未完成感**——比如分类标签的位置是一个虚线轮廓而不是实心色块，旁边有一个微妙的脉动动画（Memphis 风格的小圆点在闪）。这种视觉信号让用户的视线自然被吸引过去，但不是警报式的红色大字。
