# Moni 预算系统规格文档

## 0. 文档定位

本文件用于补齐 Moni 当前文档体系中关于预算系统的缺口。

它的目标不是替代 `DESIGN.md`，而是为正在进行集成工作的协作者提供一份可执行的预算系统规格，覆盖：

1. 表现层显示规则
2. 逻辑层计算规则
3. 持久化层数据结构建议
4. 本轮首页集成范围
5. 暂不纳入本轮实现的扩展项

本文件尤其用于回答以下问题：

1. 有预算时首页看板怎么显示
2. 无预算时首页看板怎么显示
3. 预算健康 / 警戒 / 超支三种状态分别怎么表现
4. 分类概览是否需要预算信息
5. 分类预算是否属于本轮集成范围

## 4. 本轮预算系统实施结论

### 4.1 是否有必要抽成独立文档

有必要，而且建议独立成文。

原因如下：

1. 预算系统当前既涉及首页表现层，又涉及逻辑层计算与持久化层
2. 现有 `DESIGN.md` 只覆盖预算卡视觉规则，不足以指导集成
3. `Moni_Homepage_Integration_Spec.md` 是首页集成文档，不适合把预算系统底层逻辑全部塞进去
4. 协作者如果只看现有文档，容易误判“分类预算已经要做”或者“无预算时所有预算痕迹都要消失”

### 4.2 本轮冻结实施范围

本轮预算系统只冻结以下范围：

1. 月度总预算
2. 首页看板预算卡的展示逻辑
3. 预算状态分级与 UI 表现
4. 无预算态的首页表现
5. 预算逻辑层基础计算
6. 预算持久化层最小结构
7. 预算引发的情景提示卡触发条件占位

### 4.3 本轮明确不做的内容

以下内容不纳入本轮集成实施：

1. 分类预算的正式落地
2. 分类概览上的预算刻度与超预算深色段
3. 日卡片受预算状态影响的单日染色反馈
4. 预算预测、滚动预算、跨月 carry-over
5. 预算历史版本

这些内容保留为后续扩展，不作为当前协作者的实现要求。

## 5. 预算系统状态模型

### 5.1 总预算状态分级

本轮冻结以下总预算状态：

- `none`：未设置预算
- `healthy`：预算健康
- `warning`：接近上限
- `exceeded`：已经超支

### 5.2 状态阈值

本轮建议冻结为：

1. `none`：当前账本未设置本月总预算，或预算值无效
2. `healthy`：`spent / budget < 0.7`
3. `warning`：`0.7 <= spent / budget <= 1`
4. `exceeded`：`spent / budget > 1`

说明：

1. 这个阈值与当前原型代码的 `budgetPct < 70 ? mint : budgetPct < 90 ? amber : coral` 有轻微差异
2. 当前原型里的 `90%` 警戒阈值是视觉试作，不足以作为系统规格
3. 既然你已经明确“后端预算系统还没实现”，这里更适合直接冻结逻辑层阈值

### 5.3 推荐状态字段

```json
{
  "budget": {
    "enabled": true,
    "status": "warning",
    "period": "2026-04",
    "amount": 3000,
    "spent": 2450,
    "remaining": 550,
    "usageRatio": 0.8167,
    "remainingDays": 9,
    "dailyAvailable": 61.11
  }
}
```

## 6. 表现层规格

## 6.1 首页看板卡片

### 有预算时

首页看板显示两张卡：

1. 预算卡
2. 近 7 天支出折线图卡

表现规则：

1. 预算卡位于轮播容器内
2. 顶部进度条颜色跟随预算状态
3. “剩余金额”文案跟随预算状态着色
4. “日均可用金额”在 `warning` / `exceeded` 态下允许强调

### 无预算时

首页看板只显示折线图卡：

1. 不显示预算卡
2. 不显示轮播圆点
3. 不为“无预算”强行补一张空卡
4. 不在看板区额外显示“去设置预算”按钮

如果后续需要预算引导，应通过情景提示卡承接，而不是改变看板结构。

### 不同预算状态下的预算卡表现

#### `healthy`

1. 进度条：薄荷绿
2. 剩余金额：薄荷绿
3. 卡片整体语气：轻松、正常
4. 不出现警示文案

#### `warning`

1. 进度条：琥珀橙
2. 剩余金额：琥珀橙
3. “日均可用金额”允许用警示色强调
4. 允许触发一张预算相关情景提示卡

#### `exceeded`

1. 进度条：珊瑚红
2. 剩余金额区域改为“已超 ¥X”
3. “日均可用金额”可切换为“建议日均控制 ¥X”
4. 卡片情绪明确进入警示态，但仍保持金融信息清晰，不做夸张报警

---

## 6.2 统计摘要栏

本轮预算系统不改变统计摘要栏结构。

规则如下：

1. 统计摘要仍然只显示支出 / 收入 / 笔数
2. 不在该区域插入预算字段
3. 预算相关提醒不占用统计卡片，改由预算卡和情景提示卡承担

---

## 6.3 分类概览卡

### 当前已定义部分

当前只定义了分类支出比例，不包含预算信息。

### 本轮冻结结论

本轮分类概览卡不承接预算状态。

也就是说：

1. 即便总预算已经设置，分类概览仍只展示分类支出分布
2. 即便某个分类未来存在预算概念，本轮也不在概览卡上显示预算线、超额段或警示色
3. 当前概览卡继续只服务于“钱花到哪里了”的信息表达

### 为什么这样冻结

因为“分类预算”当前还未被正式确认纳入本轮范围。  
如果现在把概览卡预算表达一起做进去，会导致协作者错误进入未冻结需求。

## 6.4 情景提示卡

预算系统可以产出的情景提示卡，当前只冻结存在性，不冻结完整引擎。

推荐触发场景：

1. 预算从 `healthy` 切换为 `warning`
2. 预算从 `warning` 切换为 `exceeded`
3. 无预算且首次使用完成导入后，提示去设置预算

推荐示例文案：

1. `本月预算已使用 82%，接下来几天可以稍微收一收`
2. `本月预算已超出 ¥230，主要来自正餐和购物`
3. `你已经有一段时间的流水了，要不要补一个月预算？`

说明：

1. 情景提示引擎本体还未定义清楚
2. 本轮只需要协作者为预算系统预留可返回提示卡的状态出口

## 6.5 日卡片

### 当前已定义

聊天沉淀里曾提出“当某天支出让预算状态从健康跳到警戒，这天卡片描边微妙变暖”的方向。

### 本轮冻结结论

本轮不实现日卡片预算染色。

原因：

1. `DESIGN.md` 当前没有正式冻结这条规则
2. 本轮首页集成重点是预算卡与逻辑层，不是再扩散日卡片表达

因此：

1. 日卡片只继续承接 AI 工作态染色
2. 不因预算状态单独改变边框颜色

## 7. 逻辑层规格

## 7.1 核心职责

预算逻辑层至少承担：

1. 读取当前账本本月预算配置
2. 计算当前月已支出
3. 计算预算状态
4. 计算剩余金额、剩余天数、日均可用
5. 生成预算相关的首页读模型字段
6. 预留预算提示卡输出能力

## 7.2 计算口径

### 周期

本轮预算周期固定为自然月。

例如：

- `2026-04-01 00:00:00` 到 `2026-04-30 23:59:59`

### 支出统计范围

本轮建议按以下口径统计预算消耗：

1. 只统计支出型交易
2. 只统计当前账本
3. 默认统计最终已落盘分类后的有效交易
4. 不统计收入
5. 不统计失败交易
6. 退款如何处理，由逻辑层沿用既有账本净值口径；若旧项目已有实现，本轮不改逻辑

### 日均可用

计算公式建议为：

`max(remaining, 0) / max(remainingDays, 1)`

当已超支时：

1. 不再展示负日均可用
2. 改为“建议日均控制”或展示为 `0`

### 与 `data range` 的关系

预算卡不跟随首页 `data range` 改变预算周期。

本轮冻结规则：

1. 首页 `data range` 影响统计摘要、分类概览、流水列表
2. 预算卡始终表达“当前自然月预算”

这点需要单独明确，否则协作者容易误把预算卡也切成“本周预算”“区间预算”。

---

## 7.3 推荐逻辑层动作

本轮只需最小预算动作：

1. `loadCurrentMonthBudget(ledgerId)`
2. `saveCurrentMonthBudget({ ledgerId, amount })`
3. `clearCurrentMonthBudget(ledgerId)`
4. `computeBudgetSummary(ledgerId, now)`

说明：

1. 预算设置入口在设置页，不在首页
2. 首页只消费预算读模型，不负责预算编辑

## 8. 持久化层建议

## 8.1 本轮最小数据结构

建议按账本存储预算配置：

```json
{
  "monthlyBudget": {
    "amount": 3000,
    "currency": "CNY",
    "updatedAt": "2026-04-07T21:30:00"
  }
}
```

如果希望与账本其他设置分离，也可以使用独立文件：

```json
{
  "ledgerId": "ledger_daily",
  "periodType": "monthly",
  "amount": 3000,
  "currency": "CNY",
  "updatedAt": "2026-04-07T21:30:00"
}
```

### 推荐结论

本轮优先追求最小实现，建议直接作为账本设置的一部分落盘，不额外拆预算专属复杂模型。

## 8.2 暂不纳入本轮的持久化内容

1. 分类预算表
2. 预算历史版本
3. 多币种
4. 周预算 / 年预算
5. 自动结转

## 9. 协作者实施清单

协作者在本轮预算系统集成中，只需要实现以下内容：

1. 逻辑层可读取当前账本本月总预算
2. 逻辑层可计算本月预算状态
3. 首页读模型返回预算卡所需字段
4. 无预算时返回 `enabled = false`
5. 首页看板按 `enabled` 决定是否显示预算卡
6. 首页看板按 `status` 决定预算卡颜色
7. 预算卡不跟随 `data range` 改变周期

明确不要求：

1. 分类预算
2. 概览卡预算线
3. 日卡片预算染色
4. 预算提示引擎完整实现
